# Release v0.1.0-paper-mvp

**Tag:** `v0.1.0-paper-mvp` (already published — do not recreate)  
**Status:** Paper MVP — interview and portfolio ready  
**Real trading:** Intentionally disabled

---

## Summary

AlphaTrade AI v0.1.0-paper-mvp is a **human-in-the-loop AI trading copilot** for crypto markets. This release delivers a complete **paper-only** workflow: market monitoring, AI-assisted analysis, deterministic risk checks, explicit human approvals, simulated execution, journaling, RAG knowledge search, usage metering, and audit trails.

This is a **decision-support and education platform**, not an auto-trading bot. No live exchange or broker connectivity is included.

---

## Main features

- **AI Trading Workspace** — LangGraph agent with guardrails, RAG context, and schema-validated responses
- **Deterministic risk engine** — 15 rules; `BLOCK` is final authority over proposals and paper execution
- **Human approval workflow** — proposals require explicit approve / reject / modify before paper orders
- **Paper execution** — simulated fills and positions; no exchange API keys for trading
- **Read-only market data** — Binance public REST or mock fallback with provenance labels
- **RAG knowledge base** — playbooks, policies, and journal lessons (not trading signals)
- **Journal → RAG loop** — trade reviews auto-sync to knowledge for future agent retrieval
- **Observability** — audit events, usage metering, organization quotas, provider status dashboard
- **Auth & tenancy** — JWT sessions, RBAC (OWNER / TRADER / VIEWER), optional httpOnly refresh cookies
- **Billing scaffold** — Stripe placeholder + usage export (`BILLING_ENABLED=false` by default)

## Safety model

| Control | Default | Meaning |
|---------|---------|---------|
| `EXECUTION_MODE` | `paper` | Only simulated execution |
| `ENABLE_REAL_TRADING` | `false` | Hard kill switch — no live orders |
| Market data | Binance **public REST** or mock | Read-only; no trading API keys |
| LLM | Optional narrative layer | Never overrides risk or approval |
| Risk `BLOCK` | Final | Blocks paper execution even if mistakenly approved |

**Real trading remains disabled by default and is not wired in this release.**

## Architecture highlights

- **Backend:** FastAPI, Pydantic v2, SQLAlchemy 2.0, LangGraph agent, structlog
- **Frontend:** Next.js 15, TypeScript, Tailwind CSS, PWA-friendly navigation
- **Data:** PostgreSQL (Docker/local), Redis (optional), Qdrant (optional; in-memory fallback)
- **Providers:** Mock by default; optional OpenAI, Qdrant, Binance public (read-only)
- **CI:** GitHub Actions — lint, tests, build, evaluation harness, optional E2E smoke

See [architecture.md](../architecture.md), [agent_workflow.md](../agent_workflow.md), and [security.md](../security.md).

## Known limitations

- No real exchange or broker execution
- No automated trading without human approval
- Stripe billing is scaffold-only — **no live charges** by default
- LLM narrative is optional; deterministic analysis + risk engine remain authoritative
- Binance public API may rate-limit; mock fallback is automatic
- Production deployment path documented but not required for this release

Full list: [limitations_roadmap.md](../limitations_roadmap.md)
